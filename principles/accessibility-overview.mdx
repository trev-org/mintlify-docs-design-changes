---

Making your game accessible expands your audience and is supported by GDK APIs, design guidelines, and Microsoft-provided testing services.

## Where to start

1. **Understand your customer** — see [Gaming and Disability Player Experience Guide](https://learn.microsoft.com/gaming/accessibility/gadpeg)
2. **Design with [Xbox Accessibility Guidelines (XAGs)](https://learn.microsoft.com/gaming/accessibility/guidelines)**
3. **Implement** — use the APIs below
4. **Test** — optionally engage [Microsoft Gaming Accessibility Testing (MGATS)](https://developer.microsoft.com/games/xbox/partner/certification-mgats)

## Development APIs and services

| Area | API / Service | Platforms |
| --- | --- | --- |
| Subtitles / captions / text display | `XAccessibility` | Windows, Xbox |
| Subtitles contrast | `XAccessibility` (`XHighContrastMode`) | Windows, Xbox |
| Communication | Azure PlayFab Party | Cross-platform |
| STT / TTS | PlayFab Party STT/TTS | Cross-platform |
| Screen narration | `SpeechAccessibility` | Windows, Xbox |

## Release management resources

| Resource | Use case |
| --- | --- |
| [**Microsoft Gaming Accessibility Testing (MGATS)**](https://developer.microsoft.com/games/xbox/partner/certification-mgats) | Optional confidential accessibility testing |

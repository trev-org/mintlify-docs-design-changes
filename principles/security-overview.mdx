For full security requirements covering the secure (Partners) GDK, dev hardware, asset custody, confidentiality, breach consequences, and audit, see [Security requirements](/gdk/security-requirements).

Key takeaways:

- The secure (Partners) GDK — including dev hardware, software, docs — must **not** be shared
- Store dev hardware in a **secured, locked location**
- Designate an **Asset Custodian**
- Don't discuss the secure GDK with anyone outside your organization
- Microsoft may audit at any time

The GDK supports multiple monetization models with both client-side APIs and back-end service integration.

## Client-side: XStore

Use `XStore*` APIs to query, advertise, and manage in-game products: consumables, durables (DLC), subscriptions, bundles, and trials.

### Fundamentals topics

- [**Basic store operations**](/publishing/xstore-basic-operations) — query catalog \+ entitlements, purchase flow
- [**Choosing the right product type**](/publishing/xstore-product-types) — bundles, consumables, durables, subscriptions
- [**Consumable-based ecosystems**](/publishing/xstore-consumables) — in-game currency best practices
- [**Bundles and Season Passes**](/publishing/xstore-bundles) — multi-product offerings
- [**Product sharing model**](/publishing/xstore-sharing-model) — how licenses share on Xbox and Windows

## Server-side: Microsoft Store Services

| Service | Purpose |
| --- | --- |
| **Collections** | Query what a user owns; fulfill consumables; detect Game Pass subscription status |
| **Clawback** | Detect and manage refunds for fulfilled consumables |
| **Purchase** | Add free items to a user's account |
| **Recurrence** | Manage user subscriptions |

<Note>
  Collections supports X-token auth (legacy), but other Store Services do **not**. Transition to **Microsoft Entra ID \+ UserStoreId** for full feature access.
</Note>

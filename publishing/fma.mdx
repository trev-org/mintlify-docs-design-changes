---
title: "Failure Mode Analysis (FMA)"
description: "How Xbox Certification scores issues for severity, probability, and repeatability."
---

**FMA** is the framework Xbox Certification uses to score issues. Three values are summed.

## Issue types

| Type | Meaning |
| --- | --- |
| **CFR — Condition for Resubmission** | Blocks ship until fixed |
| **SRI — Standard Reporting Issue** | Serious but doesn’t block publication |
| **ION — Issue of Note** | An SRI-class issue that doesn’t map to an existing test case |

## Severity

| Rating | Meaning |
| --- | --- |
| -4 | No relevant effect |
| 1 | Very minor effect |
| 2 | Minor impact |
| 6 | Moderate impact |
| 12 | Critical impact |
| 22 | Catastrophic (High Business Impact) |

## Probability

| Probability | Rating |
| --- | --- |
| Extremely unlikely (0–10%) | -3 |
| Remote (11–20%) | 2 |
| Occasional (21–40%) | 3 |
| Moderate (41–70%) | 4 |
| Frequent (71–90%) | 6 |
| Almost inevitable (91–100%) | 8 |

## Repeatability

| Repeatability | Rating |
| --- | --- |
| Won't happen a second time | -3 |
| Low | 2 |
| Moderate | 3 |
| High | 4 |
| Almost certain | 5 |
| Certain to recur | 6 |

Calculated: `(successful reproductions / total attempts) × 6`

## Final score

**Severity \+ Probability \+ Repeatability** = total. Total maps to CFR / SRI / ION.

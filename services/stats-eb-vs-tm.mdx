---
title: "Stats: event-based vs title-managed"
description: "Compare the two user-stats implementations and pick the right one for your title."
---

Xbox services has **two Stats implementations**. New titles should use **title-managed**, **except** when you need Rich Presence variables.

|  | Title-managed | Event-based |
| --- | --- | --- |
| Source of truth | Your title | The service |
| What you send | The actual stat value | Telemetry events; service runs configured rules |
| Server validation | Minimal | Configured rules |
| Drives other services | Featured Stats \+ leaderboards only | Achievements, matchmaking decisions, Rich Presence variables |
| Native "minutes played" | No — track yourself | Yes |

## Update frequency

- **Event-based** — batched, ~5–20 seconds when connected
- **Title-managed** — flushes every **5 minutes** automatically. Manual flush throttled to **30s minimum**. ~2–10 seconds when connected

## When to use which

- **Title-managed** — default for new titles
- **Event-based** — use when you need Rich Presence variables, stats that drive achievements/matchmaking, or detailed action metadata

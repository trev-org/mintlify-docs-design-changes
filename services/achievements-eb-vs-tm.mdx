title: "Achievements: event-based vs title-managed"

---

Xbox services has **two Achievements implementations**. New titles should use **title-managed**.

|  | Title-managed | Event-based |
| --- | --- | --- |
| Unlock trigger | Direct API call | Telemetry events |
| Unlock owner | The title | Xbox service |
| Configuration | Strings, images, rewards | Strings, images, rewards, unlock rules \+ stats \+ events |
| Progression | Direct API call | Indirect via events |
| RTA support | Yes | Yes |
| Challenges | Not supported | Supported |

## Title-managed requirements

1. **Cannot already have released with event-based.** No migration path exists.
2. Must use **August 2016 XDK or later**
3. Must be in a **managed dev program** (ID@Xbox, etc.)

## Offline behavior

If the device is offline when you call unlock, the API queues the request locally. It auto-syncs when connectivity returns.

## FAQ

**Can existing titles migrate?** No.

**Can a DLC use title-managed if the base game used event-based?** No — all achievements for a title must use the same service.

**How do I reset test data?** Use `XblPlayerDataReset`.

---
title: "Multiplayer Session Directory (MPSD)"
description: "Share information needed to connect players via secure JSON session documents in the Xbox cloud."
---

**MPSD** is the cloud service that stores and synchronizes a game's multiplayer system metadata. Wrapped by the `XblMultiplayer*` functions.

## Sessions

A session is identified by `XblMultiplayerSessionHandle` and stored as a secure JSON document. Each session:

- Is created and managed by titles
- Has a unique URI
- Enables connectivity between session members
- Stores game data: per-member attributes, settings, bootstrapping info
- Contains all players' XUIDs and secure device association addresses

### Session variations

| Type | Purpose |
| --- | --- |
| **Game session** | The actual gameplay session |
| **Ticket session** | Helper session tracking matchmaking state |
| **Target session** | Created during matchmaking |
| **Lobby session** | Holds invited players waiting for the game session |

## Change notifications and disconnect detection

Clients connect to MPSD via the Real-Time Activity (RTA) web socket. This connection:

- Sends "shoulder tap" notifications when sessions change
- Detects user disconnections
- Marks disconnected users inactive

## Session handles

A handle is an immutable reference to a session. Each has a handle ID (GUID) and full session reference (SCID \+ template \+ name).

### Handle types

- **Invite handle** — invite to a specific user. Create with `XblMultiplayerSendInvitesAsync`.
- **Activity handle** — binds the session to a member as their current activity. Create with `XblMultiplayerSetActivityAsync`.

## Synchronization — optimistic concurrency

MPSD resolves conflicts with optimistic concurrency \+ read-modify-write. Pass `XblMultiplayerSessionWriteMode::SynchronizedUpdate` to write methods. On HTTP 412, refresh with `XblMultiplayerGetSessionAsync` then retry.

**Xbox Manager (XBOM)** is the Windows desktop application for managing every dev kit and dev-mode retail console in your studio. Think of it as the control plane for your console hardware fleet.

## What you can do with Xbox Manager

- **Discover and group consoles** — organize kits by team, lab, build configuration, or test purpose.
- **Set the default console** — every command-line tool and Visual Studio deploy targets the active console.
- **Update system software** — install OS recoveries and updates across one console or an entire group at once.
- **Configure console settings** — sandbox, language, region, network, debug NIC, accessibility options.
- **Manage applications and files** — install, uninstall, register loose-file builds, browse the dev partition, copy files.
- **Remote control** — send gamepad input from your PC to the console, including group remote control for multi-console testing.
- **Capture media** — record screenshots and video clips from one or many consoles for bug reports.
- **Manage test accounts** — sign accounts in and out across the fleet.
- **Console Automation** — PowerShell scripting against the fleet for nightly soak tests.
- **Multiplayer testing** — coordinate matchmaking and party tests across grouped consoles.

## Why it matters

Without XBOM you would manage each console individually through command-line tools. With XBOM you can update fifty consoles, change their sandbox, and re-deploy a build in a single workflow.

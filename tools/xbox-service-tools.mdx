The GDK ships with a suite of command-line tools for working with Xbox services during development. You can access these tools in two ways: run them directly from the GDK **bin** folder after installation, or download the standalone `XboxLiveTools.zip` package from [aka.ms/xbltools](https://aka.ms/xbltools). The full source for many of these tools is available on GitHub at [github.com/Microsoft/xbox-live-developer-tools](https://github.com/Microsoft/xbox-live-developer-tools).

<Note>
  To create your own custom tooling that integrates with the same backend services, use **Microsoft.Xbox.Service.DevTools.dll**, which is included in the GDK installation and available as open-source in the [Dev Tools library repository](https://github.com/microsoft/xbox-live-developer-tools/tree/main/Microsoft.Xbox.Service.DevTools).
</Note>

## XblDevAccount.exe — Developer Account authentication

Before using tools that make authenticated service calls (such as `XblPlayerDataReset.exe` or `GlobalStorage.exe`), sign in with your Partner Center developer account. The tool caches the token for subsequent tool invocations.

```bash
# Sign in — displays a browser UI if interactive authentication is needed
XblDevAccount.exe signin --name yourname@example.com

# Sign in to a specific Microsoft Entra ID tenant
XblDevAccount.exe signin --name yourname@example.com --tenant <tenant-id>

# Sign out and clear cached tokens
XblDevAccount.exe signout
```

<Note>
  Your Partner Center account must have the **Tools Access** permission granted at the product level before you can reset player data or access other product-scoped APIs. This is set in Partner Center under **Account Settings → Users → Product-level permissions → Xbox Live → Tools access**.
</Note>

## XblPCSandbox.exe — PC Sandbox Switcher

Use `XblPCSandbox.exe` to manage the Xbox Live sandbox on your development PC. Setting or changing the sandbox requires an **elevated (Administrator) command prompt**.

```bash
# Display the current sandbox
XblPCSandbox /get

# Switch to a development sandbox
XblPCSandbox /set XDKS.1

# Switch without relaunching system apps immediately
XblPCSandbox /set XDKS.1 /noApps

# Relaunch Microsoft Store and Xbox App in sandbox mode
XblPCSandbox /launchApps

# Return to the consumer RETAIL sandbox
XblPCSandbox /retail
```

<Tip>
  You must switch back to `RETAIL` before updating the Xbox App or Gaming Services from the Microsoft Store. These apps cannot receive updates while your PC is in a development sandbox.
</Tip>

The tool automatically corrects sandbox name casing (e.g. `xdks.1` becomes `XDKS.1`).

## XblPlayerDataReset.exe — Player Data Reset

Reset achievements, leaderboards, stats, and title history for one or more test accounts in a sandbox.

```bash
# Reset by XUID (requires prior sign-in with XblDevAccount.exe)
XblPlayerDataReset.exe --scid <SCID> --sandbox <SANDBOX.0> --xuid <XUID1,XUID2>

# Reset by test account email address (prompts for password)
XblPlayerDataReset.exe --scid <SCID> --sandbox <SANDBOX.0> --user <account@xboxtest.com>

# Reset from a file (delimited list of XUIDs or emails)
XblPlayerDataReset.exe --scid <SCID> --sandbox <SANDBOX.0> --file /path/to/accounts.txt

# Use a custom delimiter (default is comma)
XblPlayerDataReset.exe --scid <SCID> --sandbox <SANDBOX.0> --xuid <X1*X2*X3> --delimiter *
```

## XblTraceAnalyzer.exe — Trace Analyzer

The Trace Analyzer is an offline tool that reviews the Xbox service calls your title made and highlights any calling pattern violations.

<Steps>
  <Step title="Capture a Fiddler trace">
    Run [Fiddler](https://www.telerik.com/fiddler) on your development machine while your GDK title is running. When you have captured the relevant service calls, save the session as a Fiddler Session file (`.saz`).
  </Step>
  <Step title="Run the analyzer">
    Point `XblTraceAnalyzer.exe` at the `.saz` file:

    ```bash
    XblTraceAnalyzer.exe -input "C:\Traces\session.saz"
    ```

    Use `-?` or `-h` to see all available command-line options.
  </Step>
  <Step title="Review the output">
    The tool prints a report of all service calls, call frequencies, and any violations it detected. Address each violation before certification submission.
  </Step>
</Steps>

## Fiddler — Web service call inspection

Fiddler can be configured to act as an HTTPS proxy for your GDK title, allowing you to inspect raw Xbox Live REST requests and responses. This is especially useful for diagnosing authentication failures, unexpected error codes, or latency spikes.

See the full GDK documentation on [Using Fiddler to inspect web service calls](https://learn.microsoft.com/en-us/gaming/gdk/docs/tools/tools-services/live-fiddler-inspect-web-calls) for setup instructions specific to GDK titles.

## Building custom tooling

If the built-in tools do not cover your workflow, you can build your own using **Microsoft.Xbox.Service.DevTools.dll**, which ships in the GDK `bin` folder and is also available as open-source:

- NuGet package: included in the GDK installation
- Source: [github.com/microsoft/xbox-live-developer-tools](https://github.com/microsoft/xbox-live-developer-tools/tree/main/Microsoft.Xbox.Service.DevTools)

The library provides the same authenticated HTTP client and service abstractions used by the built-in tools, so you can integrate Xbox service operations directly into your CI/CD pipelines.

title: "GDK and game engines"
description: "Use the GDK by itself or with engines that integrate it."
---

You can use the GDK by directly calling its C/C\+\+ APIs, or with an engine that integrates the GDK. Three integration paths:

- **GDK game template** — start a new C/C\+\+ game from scratch
- **Custom C/C\+\+ engine** — integrate the GDK manually
- **Engine with GDK integration** — Unity, Unreal, GameMaker, Godot

Download the GDK from [github.com/microsoft/GDK](https://github.com/microsoft/GDK).

## Engine guides

- [Unity](/pc-e2e/engines/unity) — Microsoft GDK API and Tools packages for Unity
- [Unreal Engine](/pc-e2e/engines/unreal) — GDK Platform Extensions and plug-ins
- [GameMaker](/pc-e2e/engines/gamemaker) — GameMaker Studio 2 GDK Extension
- [Custom C/C\+\+ engine](/pc-e2e/engines/custom-engine) — Direct GDK \+ XSAPI linking
- [Godot](/pc-e2e/engines/godot) — XBOX Godot sample (PC only)

After choosing an engine, complete [publishing in Partner Center](/publishing/overview) and add any [optional services](/pc-e2e/optional-services) your game needs.

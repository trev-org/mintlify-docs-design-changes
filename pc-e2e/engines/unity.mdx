The Microsoft GDK packages for Unity — codeveloped by Microsoft and Unity — let you target Windows x64, Xbox Series X|S, and Xbox One from the same codebase.

## Prerequisites

- Unity Editor **2021.3 or later**
- Visual Studio 2019 (16.9) or later
- Windows SDK 22000 or later
- Windows 10 (1809\+) or Windows 11
- October 2023 GDK Update 4 or later
- Valid Unity license if targeting Xbox consoles

<Warning>
  Review the [Unity Security Update Advisory](https://aka.ms/unity_security) before starting.
</Warning>

## GDK packages for Unity

| Package | Targets |
| --- | --- |
| **Microsoft GDK API package for Unity** | PC \+ console — C# wrappers for native APIs (achievements, cloud saves, leaderboards) |
| **Microsoft GDK Tools package for Unity** | PC \+ console — MicrosoftGame.config editor \+ MSIXVC packaging |
| **Microsoft GDK Tools for Xbox** | Console — packaging for Xbox submission |
| **GXDK Input System** | Console — Unity Input System support for Xbox |
| **Game Core Render Pipeline** | Console — SRP/URP/HDRP for Xbox |

Public packages for PC development are at [Microsoft GDK Package Releases](https://aka.ms/unity_gdk_packages). Console packages require Xbox Developer Program access; request via your Microsoft representative.

## Editor add-ons for console

Install **Xbox Series X|S and Xbox One Add-ons for Unity Editor** on each dev machine to enable build profiles for those targets.

## Install the packages

1. **Window → Package Manager**
2. Select **Unity Registry**
3. Search **gdk**
4. Install **Microsoft GDK API** and **Microsoft GDK Tools**

For Xbox-only packages (`.tgz`):

1. **Package Manager → \+ → Add package from tarball**
2. Select the `.tgz` file

<Note>
  Unity packages are per-project. New projects need re-installation. Xbox add-ons must be installed on each Unity Editor install.
</Note>

## Configure the project

1. **Create GDK Settings Asset** — adds GDK-specific Project Settings (one asset active at a time; can have multiple)
2. **Create a Microsoft Game Config Asset** — generates `MicrosoftGame.config`
3. **Associate the config in GDK Project Settings**
4. **Edit the config** — viewable inline in the Unity Inspector
5. **Associate with Partner Center** — use the Store Association wizard to auto-map identity values

<Tip>
  Get the Publisher value from **Package/Identity/Publisher** in Partner Center and paste it into the Publisher field in the Microsoft Game Config Editor.
</Tip>

## C# wrappers

The GDK API package exposes C# wrappers for native GDK APIs. Use them directly from your Unity scripts to call XSAPI and Gaming Runtime features.

## Samples

Three sample sets:

- **Unity Registry samples** — included with the GDK API package. Basics: Achievements, Cloud Saves, Game Saves, In-Game Store, Leaderboards, Sign-In, TTS, User. Requires Unity Input System.
- **GDK Unity samples** (from the GDK download) — advanced: `UnityAchievements`, `UnityInGameStore`, `UnityLeaderboardsTitleManaged`, `UnitySimpleMPSD`, `UnityRumble`, `UnityRumblePlayFabMultiplayer`, `UnitySimpleHttpCall`, `UnitySimpleWebSockets`, `UnitySimplePLM`, `UnityAssetWorkflow` (Xbox only)
- **Unity HandheldBestPractices sample** — Windows handheld best practices ([repo](https://aka.ms/hhexpsample_unity))

Public samples: [Xbox-GDK-Samples on GitHub](https://aka.ms/gdk_unity_samples).

## Build and package

### Windows

Configure shell visuals, localization, and packaging in **GDK Settings**. No Build Profiles changes needed. Output: MSIXVC in the MSIXVC folder.

### Console (Xbox One / Xbox Series X|S)

- **Shell Visuals Mode → Copy Only** (auto-copies Store images pre-package)
- **Project Settings → Player → Publisher Settings → Package Encryption**:
  - **Development** — sideloadable (use **Build and Run**)
  - **Submission** — Partner Center upload only; cannot sideload
- **Build Profiles**: set platform to Xbox One or Xbox Series, **Build Type = Development player** (sideload) or **Master player** (submission), **Deploy Method = Package**

Output: XVC package in the XVC folder.

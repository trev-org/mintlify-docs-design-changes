# Basic Info

The **Basic Info** page of the game submission process is where you define genre, privacy, player settings, and contact details.

The table below shows the different sections that are available for games and add-ons on the page.

| Basic info | [Genre](#genre)| [Contact and privacy](#contact-and-privacy)| [Player modes](#player-modes)|[Accessibility](#accessibility) | [Additional properties](#) |
|----------|:----:|:----:|:----:|:----:|:----:|
| Game     |   o  |   o  |   o  |   o  |  o   |
| Add-on   |      |   o  |      |      |  o   |

## Genre

A game's genre represents the category or style of gameplay that it shares with other games belonging to the genre. Your game genre will be displayed on the product details page. Gamers can search by genre to discover games in a specific genre.

You must select at least one genre keyword. You can select as many other genres as possible that are related to your game.

### Primary genre

The primary genre is the one shown prominently on the Xbox Store listings when there's space to display only one genre. If you select more than one genre keyword, select the one from the **Primary genre** dropdown that should be ranked first for display. By default, the genre that comes first in alphabetical order is selected as primary.

## Contact and privacy

Contact information and privacy settings are shown on your game's product page.

In this section, you must indicate whether your game accesses, collects, or transmits any personal information. If you answer **Yes**, you must provide a URL for a description of your privacy policy. Otherwise, the URL is optional.

**NOTE:** If we detect that your packages declare capabilities that could allow personal information to be accessed, transmitted, or collected, we will mark this question as **Yes**, and you'll be required to enter a privacy policy URL.

### Privacy policy URL

You're responsible for ensuring your game complies with privacy laws and regulations, and for providing a valid privacy policy URL here, if required.

To help you determine if your game requires a privacy policy, review the [App Developer Agreement](https://query.prod.cms.rt.microsoft.com/cms/api/am/binary/RE4OG2b) and the [Microsoft Store Policies](/windows/apps/publish/store-policies#105-personal-information).

**NOTE:** Microsoft doesn't provide a default privacy policy for your game. Likewise, your game isn't covered by any Microsoft privacy policy.

### Support contact info

Enter the URL of the web page where your customers can go for support with your game, or an email address that customers can contact for support. We recommend including this information for all submissions, so that your customers know how to get support if they need it. Note that Microsoft doesn't provide your customers with support for your game on your behalf.

### Website

Enter the URL of the web page for your game. This URL must point to a page on your own website, and not your game's web listing in the Store. The URL shouldn't specify a locale. This field is optional but recommended.

## Player modes

Indicate the support your game offers for single or multiplayer gaming.

### Single player

If your game is a single player game, or supports a single player mode, then turn on the **Single player** toggle. Select each platform that supports single player play. You must select at least one platform.

### Multiplayer

If your game can be played by more than one player at a time, then turn on the **Multiplayer** toggle. You'll be shown several multiplayer support options. Select the options and platforms applicable to your game.

Split screen: Select this option if your game allows a single screen to be split into multiple display areas and enables multiple players to simultaneously play the game by interacting with their own portion of the display area.

Cross-platform multiplayer: Select this option if your game supports multiplayer play with players on other devices or game platforms beside Xbox.

Cross-platform co-op: Select this option if your game enables players to work together toward a shared goal with players on other devices or game platforms beside Xbox.

**Require {% term xbox-live %} Gold** Select this option if your game requires an {% term xbox-live %} Gold subscription. Select only if your game is playable online mode only as players will be required to be online and connected to the Xbox network.

**Local multiplayer:** Select this option if your game supports multiple players who are connected through a local network.

**Online multiplayer:** Select this option if your game is playable in multiplayer mode when players are connected through the internet.

**Local co-op:** Select this option if your game supports co-op game play for players who are connected through a local network.

**Online co-op:** Select this option if your game supports co-op game play for players who are connected through the internet.

## Accessibility

If you have tested your game to ensure it meets accessibility guidelines, then check this box. For more information on accessibility, see Accessibility guidance.

To indicate that you have implemented specific accessibility features, navigate to the Accessibility section in the Gaming metadata page where you can specify different types of accessibility features your game supports. This will improve your game’s discovery among people who need specific kinds of accessibility support in the games they play.

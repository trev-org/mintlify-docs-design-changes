# What is PlayFab Community?

PlayFab Community is a set of cross-platform tools and services that enable game creators to facilitate engaging social experiences and foster long term player connections and communities. 

## Key Features

PlayFab Social features revolve around interactions between players, and includes friends lists, player groups, synchronous player communications, and leaderboard rankings.

### Friends

Integrate with your players' existing social networks (like Facebook, Steam, or Xbox Live) or manage a custom friends lists within your game. Any player in your title may be friends with any other player in your title. For more information about friends, see the [Friends overview](associations/friends/index.md). 

### Groups

Create durable player groupings with member roles and shared data to create customizable player groupings in games such as guild or clan experiences. These groups can be stored and retreived, allowing for long-term player grouping and group management. For more information about player grouping, see the [Groups overview](associations/groups/index.md). 

If you want ephemeral player groupings that are only needed during the runtime of the game, you may want to check out the Multiplayer [Lobby overview](../multiplayer/lobby/index.md).

### Synchronous Player Communications

Add synchronous voice and text communications into your game, powered by [PlayFab Party](../multiplayer/networking/index.md). Add flexible player comms to your game to build live communication scenarios for competitive or cooperative game modes. Ensure accessible comms with built-in speech-to-text, text-to-speech and translation features powered by Azure Cognitive Services.

For more information about in-game player communications, see the [Player Communications overview](associations/friends/index.md). 

### Leaderboards

Rank players based on their achievements and performance. Use leaderboards to rank players on any arbitrary ranking criteria, across multiple columns with column-based filtering, to build highly customizable experiences for any game. 

For more information about ranking and leaderboards, see the [Leaderboards overview](leaderboards/index.md). 

If player results do not need to be shared broadly with other players, [Player Data](../player-progression/player-data/index.md) may be better suited to store this data.

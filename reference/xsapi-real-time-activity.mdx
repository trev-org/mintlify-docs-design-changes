Real-Time Activity (RTA) is the WebSocket layer behind every change notification in XSAPI. You don't subscribe directly — features like multiplayer, social, and presence do that internally — but you handle connection state and resync events.

## Connection state

| Function | Description |
| --- | --- |
| `XblRealTimeActivityAddConnectionStateChangeHandler` / `Remove` | Subscribe to connect / disconnect events |
| `XblRealTimeActivityConnectionStateChangeHandler` | Callback signature |
| `XblRealTimeActivityAddResyncHandler` / `Remove` | Subscribe to resync (re-fetch) events |
| `XblRealTimeActivityResyncHandler` | Callback signature |
| `XblRealTimeActivitySubscriptionErrorHandler` | Subscription error callback signature |

## Types

Enums: `XblRealTimeActivityConnectionState`, `XblRealTimeActivitySubscriptionState`. Handle: `XblRealTimeActivitySubscriptionHandle`.

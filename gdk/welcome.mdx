The Microsoft Game Development Kit (GDK) is a complete set of tools, APIs, extensions, and programming models that can be used across current and future Microsoft Gaming platforms. It is the direct result of years of feedback from game development partners.

## Why the GDK?

Before creating the GDK, the Microsoft Gaming team asked developers what could be improved. Three themes came up consistently:

- **App Model friction:** The Xbox One ERA App Model and WinRT APIs forced bespoke work that didn't translate to other platforms.
- **PC parity:** UWP made it hard to ship the same Win32 game on Windows.
- **Iteration speed:** Iterative development on Xbox was held back by tooling reliability.

## What the GDK does about it

The GDK is designed so you can:

- Continue developing games the way you already do — Win32 \+ Microsoft Gaming extensions like DirectX 12.x.
- Share as much code as possible across PC, Xbox, and Xbox Game Streaming.
- Use fast, reliable, developer-focused tools and platforms.
- Adopt new multi-platform services and experiences without re-architecting.

Microsoft rewrote game-centric services — Xbox services, Connected Storage, packaging — to be identical across platforms. The result: higher code reuse, easier testing, and a clear path to ship Win32 PC games in the Microsoft Store and join initiatives like PC Game Pass.

The GDK is not a new programming model. It's a way to keep writing your game the way you've always written it — using Windows APIs and Microsoft Gaming extensions.

---
title: "Gaming Runtime glossary"
description: "Key terms used across the Microsoft GDK and Gaming Runtime."
---

| Term | Meaning |
| --- | --- |
| **Gaming platform** | OS-layer gaming components: `WINAPI_FAMILY_GAMES`, the GDK Game OS, and OneCore features. Ships as `xgameplatform.lib` (part of the Windows SDK). |
| **xgameruntime.lib / XGameRuntime.h** | Static lib \+ header compiled into the game. Entry point for Gaming Runtime features. |
| **Gaming Runtime** | Suite of components required to use Gaming Runtime features on PC. Installed and serviced as a single unit. |
| **Gaming Runtime Services (GRTS)** | Out-of-process services backing Gaming Runtime features. Delivered via the `Microsoft.GamingServices` Store package. |
| **xgameruntime.dll** | In-process DLL referenced by `xgameruntime.lib`. Manages communication with GRTS. Delivered via `Microsoft.GamingRuntime`. |
| **Game Runtime UI App** | App invoked by GRTS or `xgameruntime.dll` to show UX (e.g. TCUI) on the game's behalf. |
| **Gaming Cloud Services** | Cloud-hosted REST APIs — Xbox services, PlayFab, Microsoft Store. |
| **Gaming Runtime Packages** | Packages declared as game dependencies so the Gaming Runtime is installed before the game. |
| **Xbox Authentication Library (XAL)** | Header-only library compiled into the game; calls directly into `GameRuntime.lib`. |
| **Xbox Service API (XSAPI)** | Static libs or DLLs consumed by the game. May call into the Gaming Runtime but ships separately. |
| **Social Data Cache Service** | Console-only service that shares Xbox services social data across UX apps. Not part of the Gaming Runtime. |

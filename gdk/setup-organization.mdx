To give your team access to secure Xbox resources (documentation, downloads, forums), associate a **Microsoft Entra tenant** with your Partner Center account.

## Prerequisites

A Partner Center account that has been onboarded for Xbox development.

## Set up the tenant

1. Sign in to Partner Center as the **account owner** → Account Settings (top-right cog).
2. Open **Organization Profile → Tenants** under both **Partner** and **Developer** tabs to check if a tenant is already associated.
3. If none, click **Create Microsoft Entra ID** to create one for free under `yourOrganization.onmicrosoft.com`.
4. Create a Global Administrator account during setup.

Already have an Entra tenant? Follow [Associate Microsoft Entra ID with your Partner Center account](https://learn.microsoft.com/windows/uwp/publish/associate-azure-ad-with-partner-center).

## Manage users

In **Account settings → User management**, you have three options (sign in with a Manager \+ Global Administrator account):

### Add existing users

Users already in your Entra tenant.

1. **User management → Add existing users**
2. Select users (use search if needed)
3. Assign role(s) or custom permissions
4. Click **Save**

<Tip>
  Most organizations assign new users the **Developer** role.
</Tip>

### Create new users

Brand new accounts in your tenant \+ Partner Center.

1. **Add users → Create new users**
2. Enter first name, last name, username
3. (Optional) Make them a Global Administrator (requires password recovery email)
4. Assign group membership, roles/permissions
5. **Save** — capture the temporary password from the confirmation page (only shown once)

### Invite outside users

Email invites to users outside your tenant (added as Entra guest accounts).

1. **Add users → Invite users by email**
2. Enter up to 10 email addresses (comma/semicolon-separated)
3. Assign roles/permissions
4. **Save**

<Warning>
  Outside users can't perform certain Visual Studio tasks like Store association or package upload. For those users, use **Create new users** instead.
</Warning>

## Troubleshooting

- **Avoid nested groups.** Groups inside groups aren't supported — permissions won't apply.
- **Re-add the user.** If you suspect misconfiguration, **Delete → Delete from account**, then re-add with the correct role.

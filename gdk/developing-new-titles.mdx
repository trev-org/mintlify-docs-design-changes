---
title: "Developing a new GDK title in Visual Studio"
description: "Create a new Microsoft GDK project using Visual Studio project templates."
---

GDK games can target PC, Xbox, or both from a shared codebase. Installing the GXDK adds project templates to Visual Studio.

## Templates

| Target | Template | Use case |
| --- | --- | --- |
| Desktop | **Direct3D 12 Desktop Game (Classic)** | Standalone PC game, stock x64 MSBuild platform |
| Desktop | **Direct3D 12 Desktop Game** | Standalone PC game, `Gaming.Desktop.x64` custom platform |
| Desktop | Direct3D 12 Desktop Game Makefile | Build a PC game from a makefile |
| Desktop | Desktop Static Library | Static lib for a GDK PC game |
| Desktop | Desktop DLL | DLL for a GDK PC game |
| Console | **Direct3D 12 Xbox Game** | Standalone Xbox console game |
| Console | Direct3D 12 Xbox Game Makefile | Xbox game from a makefile |
| Console | Xbox Static Library | Static lib for a GDK Xbox game |
| Console | Xbox DLL | DLL for a GDK Xbox game |

## Create a project

1. **File → New → Project**
2. Set filters: **Language = C\+\+**, **Platform = Xbox** or **Windows**, **Project Type = Games**
3. Pick the template, click **OK**

## Picking a template

### PC only — use **Direct3D 12 Desktop Game**

- Uses `WINAPI_FAMILY_DESKTOP_APP`; full PC API surface available
- Links `kernel32.lib`, `user32.lib`, etc.
- Includes `XGameRuntime.h` and links `XGameRuntime.lib`
- XSAPI extension library available by default
- Stock x64 or `Gaming.Desktop.x64` build configuration
- Generates a `MicrosoftGameConfig.mgc` placeholder

### Xbox only — use **Direct3D 12 Xbox Game**

- Uses `WINAPI_FAMILY_GAMES`; links `XGamePlatform.lib`
- Ready for D3D12.x extensions and Xbox hardware features
- Includes `XGameRuntime.h` \+ `XGameRuntime.lib`
- XSAPI extension library available by default
- Uses `Gaming.Xbox.XboxOne.x64` MSBuild platform (set up for remote debugging on an Xbox dev kit)
- Generates a `MicrosoftGameConfig.mgc` placeholder

### Both PC and Xbox

Start with one template and add a **second build configuration**:

- Start with **Direct3D 12 Desktop Game** (default `Gaming.Desktop.x64`)
- Add a second configuration with platform `Gaming.Xbox.XboxOne.x64`
- Visual Studio builds one binary per target

Use `#define`/`#ifdef` or split platform-specific code into separate DLLs:

- PC config → stock D3D12
- Xbox config → D3D12.x

Switching the active configuration updates the debugger, linked libraries, and other settings.

The Microsoft Game Development Kit (GDK) supports two primary development paths: building games for **Xbox consoles** (Xbox Series X|S and Xbox One) and building games for **Windows PC and handheld devices**. Both paths share the same Game Core programming model, GDK Gaming Runtime APIs, and Xbox services integration — meaning a large portion of your game code can be shared between platforms. Before writing a single line of game code, make sure you meet the prerequisites below and understand which path fits your project.

## Prerequisites

Before you begin GDK development, verify that your setup meets the following requirements:

- **Windows PC** — A 64-bit installation of Windows 10 (May 2019 Update / version 1903 or later) or Windows 11. The GDK does not support 32-bit Windows or ARM-based devices for Visual Studio extensions.
- **Visual Studio 2019 or 2022** — Either edition (Community, Professional, or Enterprise) with the **Game development with C\+\+** workload installed.
- **Xbox developer program membership** — Required for console development and to access Xbox services. Register at [xbox.com/Developers](https://www.xbox.com/Developers/). PC-only titles may use the [public GDK on GitHub](https://aka.ms/gdk) without program membership.
- **30 GB of free disk space** — Required for a full GDK installation including extensions.

<Note>
  Xbox console development requires NDA program membership with Microsoft. Without it, you cannot access console-specific GDK extensions (GDKx), development kit hardware, or NDA documentation. Visit [xbox.com/Developers](https://www.xbox.com/Developers/) to learn about available programs and how to apply.
</Note>

## Development paths

The GDK gives you two distinct — but complementary — development paths. Choose the one that matches your shipping target, or follow both to build a cross-platform title.

<Tabs>
  <Tab title="Xbox Console Development">
    Xbox console development targets **Xbox Series X|S** and **Xbox One**. It requires the full **GDK with Xbox Extensions (GDKx)**, which includes console-specific toolchains, platform toolsets, and NDA-protected APIs.

    **What's different on console:**

    - Your project uses the `Gaming.Xbox.Scarlett.x64` (Series X|S) or `Gaming.Xbox.XboxOne.x64` (Xbox One) MSBuild platform toolset instead of the standard x64 toolset.
    - The Windows API family is set to `WINAPI_FAMILY_GAMES`, which restricts you to the subset of OS APIs guaranteed to be present on the Xbox Game OS. This is enforced at compile time.
    - You deploy and debug your game to a physical **Xbox Development Kit** over the network. A fast NVMe SSD and a 10 GbE network setup is recommended for efficient deployment.
    - Console-exclusive features — spatial audio, DirectStorage, cross-generation development, hardware-level memory management, and Game DVR — are only available when targeting Xbox.
    - All console submissions must pass Xbox Requirements (XRs) certification before appearing on Xbox.

    **Getting started on console:**

    1. Install the GDK with Xbox Extensions. See [Download and install the GDK](/gdk/download-install).
    2. Set up your Xbox Development Kit hardware (NDA — requires program access).
    3. Create a new project from the **Direct3D 12 Xbox Game** Visual Studio template.
    4. Configure `MicrosoftGame.config` with your Partner Center identity details.
    5. Deploy to your dev kit and iterate.

    <Tip>
      If you're shipping on both Xbox One and Xbox Series X|S, explore the [cross-generation development](/gdk/game-config) guidance to manage feature differences between hardware generations in a single codebase.
    </Tip>
  </Tab>
  <Tab title="PC & Windows Development">
    PC development targets **Windows 10/11 desktops, laptops, and handheld gaming devices** (such as Steam Deck-class PC handhelds running Windows). You use the standard **GDK** (no Xbox Extensions required) with the `Gaming.Desktop.x64` MSBuild platform.

    **What's different on PC:**

    - Your project uses the `Gaming.Desktop.x64` MSBuild platform, which targets x64 Windows with the full `WINAPI_FAMILY_DESKTOP_APP` API surface — all standard Win32 and DirectX 12 APIs are available.
    - You build and run directly on your development PC using the local Visual Studio debugger — no separate hardware required.
    - Games are packaged in the **MSIXVC** format and distributed through the **Microsoft Store** or **Xbox Game Pass for PC**.
    - The **Gaming Services Runtime** must be present on any PC running your packaged game (it is installed automatically when a player installs from the Microsoft Store).
    - PC titles can optionally integrate Xbox services — achievements, leaderboards, multiplayer, cloud saves — without needing NDA console program membership.

    **Getting started on PC:**

    1. Install the GDK. See [Download and install the GDK](/gdk/download-install).
    2. Create a new project from the **Direct3D 12 Desktop Game** Visual Studio template.
    3. Configure `MicrosoftGame.config` with your Partner Center identity.
    4. Build and run on your local PC.
    5. Package with `makepkg` and test your MSIXVC package before submitting to the Store.

    <Tip>
      If you're adding the GDK to an **existing** desktop project rather than starting fresh, follow the [Adding the GDK to an existing desktop project](https://learn.microsoft.com/en-us/gaming/gdk/docs/gdk-dev/pc-dev/overviews/gr-add-to-existing-project) guide to integrate the Gaming Runtime without disrupting your current build system.
    </Tip>
  </Tab>
</Tabs>

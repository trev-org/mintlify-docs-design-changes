`MicrosoftGame.config` is the central manifest for every GDK title. It stores game-specific configuration — your title identity, store IDs, executable list, shell visuals, and more — and is used both during local development and when you package your game for Microsoft Store submission. Rather than editing the verbose `AppXManifest.xml` directly, you work with a gaming-centric schema, and the GDK toolchain generates a valid `AppXManifest.xml` for you at build time.

## Why a new manifest schema?

Every package distributed through Microsoft Store requires a manifest. The `AppXManifest.xml` schema has grown over many years to support a broad range of application types, making it error-prone and harder to use for game developers. `MicrosoftGame.config` provides a simpler schema designed specifically for games, with validated elements and IntelliSense support in Visual Studio, reducing mistakes and improving developer productivity.

<Note>
  As of the March 2022 GDK, `configVersion` has been updated from `0` to `1` for all new titles. As of the October 2023 GDK, **all titles must use `configVersion="1"`**. Submission Validator will report a failure for any submission that still uses `configVersion="0"`.
</Note>

## Key elements

<CardGroup cols={2}>
  <Card title="Identity" icon="id-card" href="/reference/microsoftgameconfig-schema">
    Defines your title's package `Name`, `Publisher`, and `Version`. These values uniquely identify your title and are found in Partner Center under **Product Management → Product Identity**.
  </Card>

  <Card title="ShellVisuals" icon="image" href="/reference/microsoftgameconfig-schema">
    Specifies the logos, display names, splash screen, background color, and store logo that the Windows Shell and Microsoft Store use to represent your game.
  </Card>

  <Card title="ExecutableList" icon="code" href="/reference/microsoftgameconfig-schema">
    Lists every executable in your package. Each `Executable` entry specifies the file name, `TargetDeviceFamily`, application ID, and optional shell visual overrides.
  </Card>

  <Card title="StoreId / TitleId / MSAAppId" icon="store" href="/publishing/partner-center">
    `StoreId` links your build to a Partner Center product. `TitleId` and `MSAAppId` together identify your game to Xbox Live services. When `configVersion="1"`, both `TitleId` and `MSAAppId` must be specified together.
  </Card>
</CardGroup>

## Creating and editing in Visual Studio

When you create a new GDK project in Visual Studio targeting `Gaming.Xbox.XboxOne.x64`, `Gaming.Xbox.Scarlett.x64`, or `Gaming.Desktop.x64`, a `MicrosoftGameConfig.mgc` file is automatically included. At build time, Visual Studio renames it to `MicrosoftGame.config` and copies it to your output directory.

To manually add a config file to an existing project:

<Steps>
  <Step title="Add the file to your project">
    Add your `MicrosoftGame.config` as a project file and set the platform configuration to the appropriate GDK target.
  </Step>
  <Step title="Set the Item Type">
    In the file's **Properties** panel, set **Item Type** to **Microsoft Game Config** (`MGCCompile`). This tells MSBuild to treat the file as a game config and automate `.pri` generation, renaming, copying, and post-build registration.
  </Step>
  <Step title="Verify the project file">
    After setting the item type, inspect your `.vcxproj` and confirm the following item group is present:

    ```xml
    <ItemGroup>
      <MGCCompile Include="MicrosoftGame.Config" />
    </ItemGroup>
    ```
  </Step>
</Steps>

### Managing multiple config files

You can associate different `MicrosoftGameConfig.mgc` files with different build configurations, for example separate configs for debug and release, or for Xbox One versus Xbox Series X|S. Use `DefaultApplyTo` to set the default and `ApplyTo` to override per configuration:

```xml
<ItemGroup>
  <MGCCompile Include="MicrosoftGameConfig_release.mgc">
    <ApplyTo Condition="'$(Configuration)|$(Platform)'=='Release|Gaming.Xbox.XboxOne.x64'">True</ApplyTo>
  </MGCCompile>
  <MGCCompile Include="MicrosoftGameConfig_dev.mgc">
    <DefaultApplyTo>True</DefaultApplyTo>
  </MGCCompile>
</ItemGroup>
```

## The MicrosoftGame.config Editor

In addition to editing the file in Visual Studio, you can use the standalone **MicrosoftGame.config Editor** — a UI tool that provides guided authoring of `.config` files. The editor includes a **Store Association Wizard** that connects to your Partner Center project and automatically pulls in your `TitleId`, `MSAAppId`, and `StoreId`, eliminating manual copy-paste errors.

<Tip>
  The MicrosoftGame.config Editor is the recommended way to set up title identity for the first time. It validates your values against Partner Center data in real time.
</Tip>

## Launching without a MicrosoftGame.config

During early development, you can launch your game without a `MicrosoftGame.config`. On PC, double-clicking the executable works but Gaming Runtime features and title identity are unavailable. On Xbox, GDK tooling can deploy and launch loose-file builds without a config, but full GDK features, title identity, and packaging for Store submission all require a properly configured `MicrosoftGame.config`.

<Warning>
  A `MicrosoftGame.config` is required to package and submit your title to Microsoft Store. Adopt and configure yours as soon as you begin using Gaming Runtime APIs, Xbox services, or title identity features.
</Warning>
